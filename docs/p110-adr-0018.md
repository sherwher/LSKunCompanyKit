## 결정 (한 문장)

**LSKunCompanyKit plugin core 는 외부 harness (cmux / tmux / ralph / ultrawork / CI 자동화) 를 영원히 도입하지 않는다. plugin 자체가 이미 harness 다.**

## 배경

P106 메타 리뷰 (2026-05-27) 의 4 페르소나 다자 토론 — 실사용자 / Plugin·Agent 아키텍트 / 도입자·보안 / Critic — 에서 **만장일치** 로 도출된 결론. 4 round 토론을 거쳐 합의:

> "외부 harness 도입 = NO. ADR-0009 (self-contained) + ADR-0014 (시간 진화 부정) 위반.
> plugin 자체가 이미 harness — PreToolUse:Task hook + SessionStart + doctor 23 진단.
> 부족한 건 자기관찰 도구 (자기 진단 능력) 이지 외부 자동화가 아니다."

본 결정은 4회째 같은 결론 재발견을 차단하기 위해 박제. P109-A/B/C (자기관찰 도구) 완성으로 결정의 정합성 실증 완료.

## 결정 본문

### 1. Plugin core 는 외부 harness 를 도입하지 않는다

다음 도구·패턴 모두 plugin core 의존 0:

- **cmux / tmux** — 멀티 패널 워커 병렬 실행
- **OMC ralph / ultrawork** — 자기 참조 루프 / 병렬 실행 엔진
- **CI 자동화** — GitHub Actions / GitLab CI 등 plugin 자체 빌드 외 워크플로
- **외부 cron / scheduler** — 정기 자동 실행
- **AI agent framework** (LangChain / AutoGen / CrewAI 등) — plugin core 의존 0

### 2. Plugin 자체가 harness 다

본 plugin 의 다음 메커니즘이 harness 역할:

- **PreToolUse:Task hook** (ADR-0017) — dispatch 검증·차단
- **SessionStart hook** — 활성 회사 컨텍스트 자동 주입
- **doctor 진단** (24개) — 환경 health check
- **`/lskun-kit:work` 라우팅** (ADR-0004) — CPO 자동 매칭·dispatch·결재

### 3. 부족한 자기관찰 능력은 doctor 확장 + 사용자 명시 view 로 보완

자동 평가·KPI·대시보드는 ADR-0006 정신 유지하되, **사용자 명시 view 는 허용**:

- `/org --usage` (P109-A) — 워커별 dispatch count + last_seen
- `/lskun-kit:audit-rotate` (P109-B) — audit log 회전 명령
- doctor [27]~[30] (P110-B) — 30일+ 미사용 / 빈 도메인 / hired 오염 / archived dangling

**Boundary** (ADR-0006 정신 + P109-A 박제):

| 금지 (불변) | 허용 (도입 가능) |
|---|---|
| 자동 평가 / 점수화 | 사실 표시 (dispatch count) |
| 정기 자동 산출 (월간/분기) | 사용자 명시 옵션 (`--usage`) |
| 외부 자동 전송 | 로컬 stdout 만 |
| 워커 자동 해고·평가 | 사용자가 보고 판단 |
| KPI / 대시보드 / 시각화 | audit log 의 단순 view |

### 4. Escape hatch 없음

외부 harness 가 필요하면 사용자가 **별도 add-on** 으로 자유롭게 사용 (plugin core 만 미도입). 본 plugin 의 add-on 정책 = ADR-0009 (self-contained default) 와 동일.

### 5. 재도입 조건

5회째 같은 결론 ("no harness") 을 재발견하면 **그때 새 ADR + 5회 재발 증거** 박제 필수. P106 (1회) → P110-A 박제 (2회 잠재) — 임계점까지 3회 남음.

## ADR 정합

| ADR | 관계 |
|---|---|
| ADR-0009 (self-contained) | **강화** — 외부 SDK / harness 모두 plugin core 미도입 |
| ADR-0014 (reflection 폐기) | **연장** — "시간 흐름으로 자동 진화" 부정의 일관 |
| ADR-0006 (audit 불변) | **유지** — 자동 평가 금지, 사용자 명시 view 만 허용 |
| ADR-0002 §5 (KPI 금지) | **유지** — 평가·점수·랭킹 영구 금지 |

## 검증 (P109 실증)

- `/org --usage` 출시 후 사용자 자기관찰 수단 확보 — 외부 harness 불필요 입증
- `/lskun-kit:audit-rotate` — 데이터 무한 누적도 사용자 명시 명령으로 처리, 자동 cron 불필요
- CLAUDE.md slim 47KB → 11.8KB — 외부 메모리 시스템 없이도 컨텍스트 비용 관리 가능

## 폐기·금지 (영구)

- plugin core 의 cmux / tmux / ralph / ultrawork / CI 자동화 SDK 직접 사용
- doctor 의 자동 회수 (cron 같은 정기 실행) — 사용자 명시 호출만
- `/org --usage` 의 자동 산출 (보고서 정기 생성) — 옵션 명시 호출만
- audit log 의 자동 회전 — `/lskun-kit:audit-rotate --execute` 명시만
- LLM-as-judge 자동 평가 — ADR-0014 폐기 결정과 일관, 영구 금지
